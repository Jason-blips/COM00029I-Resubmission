### How to Use

#### 1. Clone the Repository
```bash
git clone <your-repository-url>
cd "SYS2 Re"
```

#### 2. Review by Question Number
1. Open a question paper (for example, `Assessed Questions/Q3.pdf`).
2. Extract the corresponding submission package (`Coursework/Q3.zip`).
3. Compare your implementation against requirements.
4. Cross-check with comments in `feedback.md`.

#### 3. Track Improvements
1. Keep a note file for each question (for example, `notes/Q3.md`).
2. Record:
   - what was incorrect,
   - why it happened,
   - how to avoid it next time.

### Recommended Workflow
- **Step 1:** Read the official question PDF.
- **Step 2:** Inspect your submitted ZIP content.
- **Step 3:** Review matching feedback section.
- **Step 4:** Summarize key takeaways in personal notes.
- **Step 5:** If needed, rebuild or re-test locally with corrected logic.

### Notes
- Use a PDF reader plus an archive tool (7-Zip/WinRAR/system unzip).
- Keep naming consistent (`Q1` ... `Q9`) for easy cross-reference.
- Avoid committing extracted temporary folders unless they are intentionally organized for revision.

### 使用方法

#### 1. 克隆仓库
```bash
git clone <你的仓库地址>
cd "SYS2 Re"
```

#### 2. 按题号复习
1. 打开对应题目 PDF（例如 `Assessed Questions/Q3.pdf`）。
2. 解压对应提交文件（例如 `Coursework/Q3.zip`）。
3. 对照题目要求检查提交内容。

#### 3. 记录改进点
1. 可为每题建立笔记（例如 `notes/Q3.md`）。
2. 建议记录：
   - 问题出现在哪里，
   - 原因是什么，
   - 下次如何避免。

### 推荐复习流程
- **步骤 1：** 阅读题目原文。
- **步骤 2：** 检查当时提交内容。
- **步骤 3：** 对照批改反馈定位问题。
- **步骤 4：** 总结知识点与改进动作。
- **步骤 5：** 必要时重新实现并本地验证。

### 注意事项
- 建议使用 PDF 阅读器与压缩工具（7-Zip、WinRAR 或系统自带工具）。
- 保持 `Q1` 到 `Q9` 命名一致，便于快速定位。
- 非必要情况下，不建议提交临时解压目录，避免仓库变杂。
